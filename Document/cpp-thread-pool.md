## 进程池相关知识

进程池作为五大池之一（内存池、连接池、进程池、协程池），其应用非常广泛，不管是客户端程序，还是后台服务程序，都是提高业务处理能力的必备模块。有很多开源的线程池实现，虽然各自接口使用上稍有区别，但是其核心实现原理都是基本相同的。

### 并发和并行

CPU 时间片（CPU Time Slice）是操作系统分配给每个可运行进程的一小段连续 CPU 执行时间，即 CPU 把自己的时间切成无数个微小的时间片，轮流分配给不同的进程使用。

**并发：**单核上，多个线程占用不同的 CPU 时间片，物理上还是串行执行的，但是由于每个线程占用的 CPU 时间
片非常短（比如 10ms），看起来就像是多个线程都在共同执行一样，这样的场景称作并发（concurrent）。注意：并不是所有应用场景都适用并发。

![](../../Git/image/fc6e6e86-2e02-49a8-9913-ca492291bc8e-1779171750411-2.png)

**并行：**在多核或者多 CPU 上，多个线程是在真正的同时执行，这样的场景称作并行（parallel）。

### 多线程的优势

多线程程序一定就好吗？不一定，要看具体的应用场景

**IO 密集型：**程序里面指令的指向，设计一些 IO 操作，比如设备，文件，网络操作（等待客户端的连接，IO 操作是可以把程序阻塞住的，此时分配给这样程序 CPU 时间片，CPU 相当于空闲下来了）；无论是 CPU 单核、CPU 多核、多 CPU，都是比较适合多线程程序的。

**CPU 密集型：**程序里面的指令都是做计算用的；CPU 单核：多线程存在上下文切换（当前线程调度完后，CPU 寄存器信息保存到线程栈上，进行下一个线程，当右轮到该任务执行，把线程栈上的信息回复到 CPU 里面），是额外的花销，线程越多上下文切换所花费的额外时间也越多，倒不如一个线程一直进行计算；CPU 多核、多 CPU：多个线程可以并行执行，对 CPU 利用率好。

### 线程

为了完成任务，创建很多的线程可以吗？线程真的是越多越好？

**线程的消耗：**

1.线程的创建和销毁都是非常"重"（耗时多，耗性能多）的操作，所以在服务执行过程中避免创建大量线程

2.线程栈本身占用大量内存，且所有的线程，共享进程的地址空间，所以一个进程所创建的线程个数有上限

3.线程的上下文切换要占用大量时间，线程过多，上下文切换花费的 CPU 时间也特别多，CPU 利用率低

4.大量线程同时唤醒会使系统经常出现锯齿状负载或者瞬间负载量很大导致宕机

![](../../Git/image/2026_5_19_15_58-1779177844221-8.png)

线程的数量一般按照当前 CPU 核心的数量来确定

### 线程池的优势

操作系统上创建线程和销毁线程都是很"重"的操作，耗时耗性能都比较多，那么在服务执行的过程中，如果业务量比较大，实时的去创建线程、执行业务、业务完成后销毁线程，那么会导致系统的实时性能降低，业务的处理能力也会降低。

线程池的优势就是（每个池都有自己的优势），在服务进程启动之初，就事先创建好线程池里面的线程，当业务流量到来时需要分配线程，直接从线程池中获取一个空闲线程执行 task 任务即可，task 执完成后，也不用释放线程，而是把线程归还到线程池中继续给后续的 task 提供服务。

### 线程池的模式

**fixed 模式线程池：**线程池里面的线程个数是固定不变的，一般是 ThreadPool 创建时根据当前机器的 CPU 核心数量进行指定。

**cached 模式线程池：**线程池里面的线程个数是可动态增长的，根据任务的数量动态的增加线程的数量，但是会设置一个线程数量的阈值，任务处理完成，如果动态增长的线程空闲了 60s 还没有处理其它任务，那么关闭线程，保持池中最初数量的线程即可。

### 线程同步

一段代码能否在多线程环境下执行，就是看这段代码是否存在竞态条件（代码片段在多线程环境下，随着线程的调度顺序不同，而得到不同的运行结果），如果在多线程环境下不存在竞态条件，称为可重入，反之称为不可重入。

**线程互斥**

互斥锁 mutex：lock_guard、unique_lock

atomic 原子类型与 CAS 操作

**线程通信**

条件变量 condition_variable：需结合互斥锁 mutex 来使用，典型例子是生产消费者模型

信号量 semaphore：看作资源计数没有限制的 mutex 互斥锁（mutex 资源计数只能是 0 或 1），mutex 只能是那个线程获取锁，由那个线程放缩，信号量 semaphore 不同，sem.wait() 和 sem.post() 可以处在不同的线程中调用。信号量 semaphore 常用于线程依赖中使用。
